# Contributing to CTTAF

We welcome contributions from researchers, theologians, software engineers, and Christian community members!

## How to Contribute

### 1. Reporting Issues
- **Dataset Issues**: Report theological inaccuracies, bias concerns, or domain problems
- **Technical Issues**: Report bugs in evaluation scripts or documentation gaps
- **Judge Performance**: Report concerning judge outputs or scoring anomalies

### 2. Contributing Questions
If you'd like to add theological questions to the benchmark:
1. Follow the question design process (see `appendices/question_design_process.md`)
2. Ensure coverage across triage levels and doctrinal dimensions
3. Include metadata: source, denomination context, expected answer guidelines
4. Submit a PR with updated CSVs and supporting documentation

### 3. Improving the Rubric
- Suggest clarifications to scoring dimensions
- Report scoring edge cases where judges disagree
- Propose new evaluation dimensions based on research needs

### 4. Code & Tooling
- Optimize evaluation scripts for speed or accuracy
- Add support for additional LLM providers
- Improve inter-judge agreement validation

## Pull Request Process
1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-contribution`
3. Make your changes with clear commit messages
4. Include documentation and tests where applicable
5. Submit a pull request with a description of changes

## Code of Conduct
We are committed to providing a welcoming environment. All contributors must:
- Respect diverse theological traditions within Christianity
- Treat all participants with dignity and respect
- Avoid dismissive language toward any denomination or perspective
- Focus on research quality and fairness

## Questions?
Open an issue or reach out to the maintainers for guidance.
